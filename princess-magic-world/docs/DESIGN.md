# 設計ドキュメント — プリンセスと ひかりの おうこく

## 1. コンセプト

「魔法の国のプリンセス」の世界観を、**4歳児がひとりで安心してあそべる**インタラクティブな 3D 箱庭として構築する。

### 4歳児向けデザイン原則

1. **文字が読めなくてもあそべる** — UI はすべて絵文字アイコン。ヒントは最初の1回だけ
2. **失敗が存在しない** — 敵・タイマー・ゲームオーバーなし。すべてのタップが「ごほうび」
3. **ワンタップ操作のみ** — ドラッグ・スワイプ・長押し・マルチタッチ不要
4. **大きなタッチターゲット** — ボタンは 66px、なかまの当たり判定は見た目より大きい球
5. **即時フィードバック** — タップの 100ms 以内に光・音・動きのどれかが必ず返る
6. **祖父母の端末でも動く** — 画質自動調整で古い iPad でも 30fps を維持

## 2. アーキテクチャ

```
main.js (Game)
 ├─ ctx = { scene, camera, renderer, audio, particles, interactables, colliders, groundMesh }
 │        ← 全モジュールで共有するコンテキストオブジェクト
 ├─ SkyAndLights   空ドームシェーダー・平行光源/半球光源・霧・雲・昼夕夜遷移
 ├─ World          地形(頂点変位+頂点カラー)・お城・池(水シェーダー)・植生・虹
 ├─ Princess       レイス(Lathe)ドレス・歩行/ターンアニメ・衣装替え
 ├─ Creatures      ユニコーン(追従AI)・うさぎ(ホップAI)・鳥(周回)・蝶(徘徊+磁力)・あひる
 ├─ Collectibles   星/ハートのスポーン・自動回収・コンボ音階・おいわい判定
 ├─ Particles      プール式パーティクル(共有ジオメトリ+カスタムシェーダー)
 ├─ KidsAudio      WebAudio 手続き生成(オルゴールワルツ+効果音)
 ├─ Input          ポインター→レイキャスト(球判定→地面判定の2段階)
 └─ UI             DOM オーバーレイ(HUD/ボタン/タイトル/バナー)
```

### 依存方向

各モジュールは `ctx` のみに依存し、相互参照しない。イベントはコールバック
(`collectibles.onStar` など)で main に集約する。

## 3. レンダリングパイプライン

| 要素 | 実装 |
|---|---|
| トーンマッピング | ACESFilmic, exposure 1.08 |
| ポストプロセス | EffectComposer → RenderPass → **UnrealBloomPass**(threshold 0.82) → OutputPass |
| 影 | PCFSoft 2048px 平行光源1灯（画質 tier 3 で無効化） |
| 空 | 背面球体 + カスタム GLSL（3色グラデ・太陽・月・ハッシュグリッドの星・夕焼けグロー） |
| 水 | カスタム GLSL（頂点波 + sin 干渉の擬似コースティクス + 縁のフェード） |
| パーティクル | Points + カスタムシェーダー（頂点属性で個別サイズ/アルファ/色、テクスチャは Canvas 生成） |

### 発光(Bloom)対象の設計

emissiveIntensity を 1.0 超に設定したものだけがブルームで「本当に光る」:
ティアラの宝石 / ユニコーンの角 / お城の窓(夜 3.1) / クリスタル(夜 ~1.8) / 星・ハート。
夜はブルーム強度自体も 0.45 → 0.80 へ上げ、ドラマチックに。

## 4. 昼・夕・夜システム

`PHASES` テーブル(空3色・霧・光源色/強度・太陽位置・夜係数・虹の濃さ)を持ち、
現在値を指数減衰 `lerp(cur, target, 1-e^(-1.6dt))` で毎フレーム遷移。
係数 1 つ(`nightFactor`)を World / Creatures に配って、窓明かり・ホタル・
きのこ発光・光る蝶・水の色まで連動させる。

## 5. パフォーマンス戦略

- ドローコール削減: 花170本・草380本を `InstancedMesh` 化、パーティクルはプール毎に 1 Points
- 毎フレームのオブジェクト生成ゼロ(THREE.Color/Vector3 をキャッシュ)
- **自動画質調整**: dt>45ms が 2.5 秒続くごとに tier↓
  (pixelRatio 2→1.5→1.2→1 / bloom OFF / shadow OFF)。`?quality=` で固定可
- SwiftShader(CI) でもエラーなく動作することをヘッドレステストで確認

## 6. オーディオ設計（外部ファイル 0 個）

- **BGM**: 92BPM・3拍子のオリジナルワルツ。オルゴール音色(基音 sine + 4倍音、鋭アタック・長減衰)
  + フィードバックディレイの残響。8小節フレーズ(C-G-Am-F-C-F-G-C)をルックアヘッドスケジューリング
- **効果音**: 星=ペンタトニックで上がるチャイム(コンボ連動) / ユニコーン=ハープグリス /
  花火=バンドパスノイズ+クラックル / あひる=sawtooth 2連 など、すべてオシレーター合成
- iOS の自動再生制限: タイトルの「はじめる」タップで `AudioContext` を初期化

## 7. 入力とインタラクション

タップ判定は 2 段階:
1. **なかま/星** — レイと当たり判定球の距離判定(メッシュ交差より高速で、見た目より大きく取れる)
2. **地面** — 地形メッシュとのレイキャスト交差点へ `walkTo`

移動はコライダー(円)押し出し + スタック検知(0.5 秒進めなければ停止)で、
池やお城にめり込まない。カメラは縦横比で距離/FOV を補間し、縦持ちでも横持ちでも
プリンセスとお城が気持ちよく収まる。

## 8. 面白さの積み上げ（モリモリ要素一覧）

収集(星→花火の週目標) / 育成感(累計カウント永続化) / 変身(ドレス5色) /
ふれあい(なかま5種) / 環境変化(昼夕夜) / 魔法(星の雨+蝶の集合) /
発見(クリスタル・きのこ・ホタル・虹・桜吹雪) / 音楽(オルゴール+コンボ音階) /
おいわい(バナー+ファンファーレ+9連花火+ツイル)
