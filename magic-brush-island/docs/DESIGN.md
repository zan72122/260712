# DESIGN — まほうのふで と いろのしま

## コンセプト

エピックミッキーの中核体験 —「**色を失った世界に、筆で命を吹き込む**」— を、
4歳児向けに再設計したもの。原作の「シンナーで消す」「敵と戦う」という緊張要素は
すべて取り除き、**回復と祝祭だけ**を残した。

### 4歳児向け設計の原則

1. **失敗が存在しない。** 落下ダメージなし、敵なし、時間制限なし、まちがった操作なし。
2. **すべての操作が何かうれしいことを起こす。** 歩くだけで色がつく。タップは
   ねらいが外れても近くのグレーのものに当たる（fat-finger 補正 4.5m）。
   何もない地面をタップしても、えのぐの splat が出て音が鳴る。
3. **文字が読めなくても遊べる。** 進行はすべて視覚と音で伝わる。テキストは
   ひらがなのみ、読み聞かせ用。
4. **報酬は「即時・小」と「蓄積・大」の二層。**
   - 即時: 1プロップぬるごとに 色の開花 + パーティクル + 3音アルペジオ
   - 蓄積: ゾーン完成 → 地面に色の洪水 + 虹 + ファンファーレ + 花火、
     全島完成 → 夕焼け + 花火大会 + 常時レインボーモード

## ペイントシステム（技術）

### プロップ
- `Mat.makePaintable(group, zone)` がグループ内の全マテリアルを clone し、
  目標色を保存してから**鉛筆スケッチ色**（輝度 + 青灰色 tint、0.7倍暗）に落とす。
- `Mat.paint(entry)` で 0.8 秒の tween 開始:
  色は 1.6 倍速で先行して戻り、スケールが easeOutBack で ~125% までぽよんと弾む。

### 地面（ゾーン洪水）
- 地形と草は `MeshToonMaterial` を `onBeforeCompile` でパッチし、
  `uZones[8]`（x, z, animatedRadius）と `uGlobalPaint` を注入。
- fragment で各ゾーンの円形マスク（sin ワブルで手描き風の縁）を合成し、
  `mix(グレー, 頂点色, mask)`。ゾーン完成時に radius を 0 → 1.75R へ 3.2 秒で拡大。
- フィナーレ後は `uGlobalPaint` → 1 で島全体が飽和。

### 色管理
Three.js r149 の legacy color mode のまま、トゥーンマテリアルの色のみ
`convertSRGBToLinear()` して sRGB 出力段でちょうど指定 hex が出るようにした
（カスタムシェーダー（空・海）はエンコード段を通らないので生値のまま調整）。

## 世界の構成

- 地形: 解析的ハイトフィールド `terrainH(x,z)`（ガウス丘 + 正弦ノイズ + 放射状の
  海への落ち込み）。メッシュ・キャラ・動物・プロップ配置が同じ関数を共有。
- 6ゾーン: おはなばたけ 🌸 / きのこのもり 🍄 / みなとまち 🏠 / もりのくに 🌳 /
  おしろのおか 🏰 / ゆうえんち 🎠。計 75 のペイント対象。
- 環境演出: シェーダー空（3モード: 昼 / 夕焼け / スーパーデイ をフィナーレで遷移）、
  波・グリッター・浜の泡リング付きの海、流れる雲、浮島 ×3、周回するウミガメ、
  旋回する小鳥、中央のまほうの泉（虹色に光る）。
- 動物 AI: ぬられるまで凍って小さく震える → ぬると家の周りをホップ、
  プレイヤーが近づくと向き直ってハートを出す。あひるは池を周遊。

## 入力

- **フローティングジョイスティック**: 画面のどこでもタッチ開始点が基準になる
  （固定位置のジョイスティックは幼児には難しいため）。
- **タップ**（<260ms・移動<14px）: レイキャスト → ペイント対象 or 地面。
  対象が painted / 不在なら hit 点から最寄りの未ペイント対象を 4.5m 以内で検索。
- キーボード（矢印/WASD + Space）は開発・PC用に併設。

## 音

すべて Web Audio でリアルタイム合成（音声ファイルなし）:
- 基音はすべて **A メジャーペンタトニック** — どのタイミングで何が重なっても
  不協和にならない。
- BGM: 92BPM のオルゴール風 8小節ループ（lookahead スケジューラ）。
  進行度 45% で対旋律レイヤーが追加される。
- SFX: ペイント（3音上昇）、星（取るたび音階が上がる）、ゾーン完成ファンファーレ、
  花火、ジャンプ/着地、動物のホップ。

## パフォーマンス

- 実測: 約 420 draw calls / 10.5万 triangles（iPhone の Safari で 60fps 圏内）。
- 草 2,400 本は 1 draw call の InstancedMesh、揺れは頂点シェーダー。
- パーティクルは 3 つの固定長プール（soft 700 / glow 1000 / heart 48）を
  リングバッファで再利用。
- pixelRatio は 2 でクランプ。影は 2048px PCFSoft 1枚。

## 検証

Playwright + ヘッドレス Chromium (SwiftShader) で自動検証:
コンソールエラーゼロ、ジョイスティック移動・ジャンプ・タップペイント・
ゾーン完成 → フィナーレの全パスを実行済み。縦・横両ビューポートで確認。
